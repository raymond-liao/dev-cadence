# 01-requirements.md

## 目的

`01-requirements.md` 记录目标、范围、非目标、验收标准、约束、歧义、Human decisions 和 G1 Requirements readiness。

## 写入阶段

`requirements`。

## 写入者

Planner，在 Supervisor 控制下完成。

## 必要输入

- [00-brief.md](00-brief.md)
- 需要时的用户澄清
- 相关项目文档、specs 或 issue context

## 记录内容

- scope 和 non-goals
- users or stakeholders
- acceptance criteria
- assumptions 和 open questions
- ambiguity analysis
- Requirements Readiness Check
- Gate G1 record

## Gate 影响

只有 expected behavior、scope、non-goals、acceptance criteria 和 verification approach 清楚到足以实现和 review 时，G1 才能通过。

## 如何阅读

实现前先看这个文件，确认什么在范围内、什么明确不做，以及是否还有阻塞问题。

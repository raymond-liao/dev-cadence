# S-025 Bug RED/GREEN 证据

## 基本信息

- ID：`S-025`
- Version：`1`
- Status：`Draft`
- Priority：`P2`
- Change Type：Feature

## 目标

为 Bug 修复建立稳定 proof ID，将修复前失败、关联修改和修复后通过证据闭环；无法直接复现时记录替代因果证据和原因。

## 背景

仅记录最终测试通过无法证明修复针对原始缺陷。稳定的 RED/GREEN 证据可以连接问题复现、修复边界和验证结果，同时允许不可直接复现的问题使用明确的替代证据。

## ✅ 范围

- 为每组缺陷证明分配稳定 proof ID。
- 存在自动测试或可重复检查时记录修复前失败和修复后通过。
- 将 proof ID 关联到相关实现修改和回归验证。
- 无法直接复现时记录替代因果证据、限制和原因。

## ❌ 非范围

- 不要求所有 Bug 都必须新增自动化测试。
- 不用推测性说明替代可获得的复现证据。
- 不在本 Story 中定义根因诊断门禁。

## 验收标准

1. 可复现 Bug 的 RED 与 GREEN 证据使用同一稳定 proof ID。
2. proof ID 能追溯到修复修改和最终验证。
3. 不可直接复现时明确记录替代证据及其局限。

## 依赖

- Bug 诊断必须先形成足够明确的问题边界。

# Output Template

## Final Report Structure

The final report should use this structure:

```markdown
# AI Native Development Framework 系统调研报告

## 1. 执行摘要

## 2. 当前方案画像

## 3. 核心判断验证

## 4. 与软件研发实践的对齐分析

## 5. 多 Agent 协作架构评估

## 6. Team Skill 交付形态设计

## 7. MVP 范围与验证计划

## 8. 风险、失败模式与质量门禁

## 9. 对 README 方案的修订建议

## 10. 下一步行动

## References
```

## Writing Style

- Write in Chinese.
- Use direct, engineering-oriented language.
- Avoid promotional wording.
- State tradeoffs explicitly.
- Use tables where comparison is clearer than prose.
- Keep recommendations concrete enough to apply to the README proposal.

## Required Sections

The final report must include:

- A prioritized list of proposal improvements.
- A clear MVP definition.
- A team Skill content map.
- A risk matrix.
- An evaluation metric set.
- A list of open questions for human decision.

## Quality Checklist

- [ ] The report answers all core research questions.
- [ ] The report cites source IDs and analysis IDs.
- [ ] The report distinguishes confirmed conclusions from assumptions.
- [ ] The report includes actionable recommendations.
- [ ] The report does not claim external validation if external research was not actually performed.
- [ ] The report is consistent with the current root README proposal.

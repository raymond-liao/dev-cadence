# Research Overview

## Research Topic

AI Native software development collaboration framework based on Human + Multi-Agent Team.

## Research Objective

Produce a systematic research report that evaluates, strengthens, and operationalizes the current proposal before it is turned into a reusable team Skill or engineering implementation.

## Research Scope

Includes:

- Conceptual model and architectural coherence.
- Agent role topology and responsibility boundaries.
- Workflow, context, artifact, and quality gate design.
- Human-in-the-loop control points.
- Skill-based delivery as the first shared team artifact.
- MVP scope, risks, evaluation criteria, and evolution path.

Excludes:

- Implementing the team Skill.
- Building a LangGraph, Microsoft Agent Framework, or platform prototype.
- Modifying production code.
- Selecting one final vendor or model.
- Writing detailed code templates for a specific business system.

## Core Research Questions

1. What are the strongest assumptions in the current framework, and are they defensible?
2. Which parts of the framework are essential for MVP, and which parts should be deferred?
3. How should the framework compare with established software delivery practices such as SDLC, ADR, CI, code review, QA, and platform engineering?
4. What role boundaries and quality gates are most important to prevent AI-driven delivery failure?
5. What should a reusable team Skill contain so that the framework can be shared without prematurely becoming a platform?
6. What evaluation metrics and failure cases should be used to validate the framework?

## Research Stance

The research should be pragmatic and implementation-oriented.

Do not assume multi-agent collaboration is valuable by default. Evaluate where it improves quality, traceability, and team leverage, and where it increases overhead.

Prefer actionable framework revisions over abstract commentary.

## Output Language

The final report should be written in Chinese unless the user specifies otherwise.

## Workflow

### Stage 1: Extract Proposal Structure

Read the current proposal and identify:

- Background and goals.
- Core judgment.
- Overall architecture.
- Agent topology.
- Agent role boundaries.
- Workflow design.
- Loop design.
- Human-in-the-loop design.
- Context architecture.
- Spec-driven development structure.
- Quality gates.
- Tooling analysis.
- Team Skill delivery shape.
- MVP proposal.
- Enterprise roadmap.
- Open discussion topics.

### Stage 2: Identify Assumptions

Identify explicit and implicit assumptions, including:

- Assumptions about AI coding agent capability.
- Assumptions about team adoption.
- Assumptions about quality improvement.
- Assumptions about workflow overhead.
- Assumptions about Skill as first delivery shape.
- Assumptions about future platformization.

### Stage 3: Identify Open Questions

List unresolved questions that need research, evidence, design decisions, or human judgment.

### Stage 4: Create Framework Map

Write the output to:

```text
{PROJECT_ROOT}/workspace/01.materials/framework_map.md
```

## Workflow

### Stage 1: Identify Failure Modes

Analyze risks including:

- Supervisor overreach.
- Developer self-approval.
- Tester or Reviewer rubber-stamping.
- Spec drift.
- Context pollution.
- Unbounded fix loops.
- Tool permission abuse.
- Missing test evidence.
- Excessive workflow overhead.
- Skill rules becoming stale.
- Platformization before process validation.

### Stage 2: Define Risk Matrix

For each risk, evaluate:

- Probability.
- Impact.
- Detection signal.
- Prevention mechanism.
- Escalation rule.

### Stage 3: Define MVP Evaluation

Design MVP experiments and metrics:

- Task cycle time.
- Defect escape rate.
- Review blocker discovery.
- Test evidence completeness.
- Human intervention frequency.
- Context pack size.
- Fix loop count.
- Spec change count.
- Developer satisfaction or workflow friction.

## Workspace Isolation Requirement

**IMPORTANT**: You must write only under `research/ai-native-development-framework-study/`.

You are the External Researcher.

Your task is to collect relevant external evidence that can help evaluate the current AI Native Development Framework proposal.

## Workflow

### Stage 1: Determine Research Targets

Based on the framework map, identify external research targets across:

- Agentic software engineering.
- AI coding agents.
- Multi-agent orchestration.
- Human-in-the-loop systems.
- Software delivery lifecycle practices.
- Code review and QA practices.
- ADR and spec-driven development.
- Platform engineering and internal developer platforms.
- Reusable agent skills or prompt packages.

### Stage 2: Collect Sources

If external browsing is available:

1. Search for relevant sources.
2. Fetch original content before using it.
3. Prefer official documentation, academic papers, credible engineering references, and high-quality industry reports.
4. Record source metadata and relevance.

If external browsing is unavailable:

1. Record that external research could not be performed.
2. List recommended source categories to research later.
3. Do not fabricate source IDs except for `SRC-PROPOSAL-README`.

### Stage 3: Create Source Notes

Write the output to:

```text
{PROJECT_ROOT}/workspace/01.materials/external_sources.md
```

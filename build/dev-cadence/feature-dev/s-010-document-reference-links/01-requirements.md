- Backlog：[当前可并行实施表](../../../../docs/delivery/backlog.md#当前可并行实施表)

## ✅ 确认范围

- 在共享 `document-conventions` skill 中集中定义选择性文档引用规则；入口和各 workflow 只负责读取、遵守和对称接入，不复制完整契约。
- 已存在、面向阅读导航且生命周期兼容的仓库内文档引用使用有意义文本和相对当前来源文档的 Markdown 链接。
- 只有真实稳定的章节锚点才可用于章节定位；不能可靠确认锚点时链接文档本身。
- Manifest 和阶段记录在同时需要阅读导航与审计身份时，必须同时保留快捷链接和精确仓库相对路径。
- 未创建目标保留明确的 `pending` 状态和计划路径，创建并验证存在后才转换为链接。
- `docs/` 长期文档不得把 `build/` 临时运行记录作为权威来源；运行记录可以链接同一 run 的已存在产物和 `docs/` 长期文档。
- 文件移动或重命名后更新受影响链接；提交前检查 tracked Markdown，workflow 完成前检查当前 run 已生成文档的本地链接。
- 对称更新 feature-dev、bug-fix、refactor 和 discovery 的共享规范接入，并以契约测试保护规则所有权、选择性链接、路径身份、生命周期和禁止 URI。

## ❌ 非目标

- 不机械转换所有文件路径，不批量重写无阅读导航价值的历史路径数据。
- 不为尚不存在的产物创建失效链接，不以链接替代审计需要的精确路径。
- 不使用本机绝对路径、`file://`、编辑器 URI 或其他不可移植引用。
- 不实现文档搜索、站点生成器或链接预览界面。
- 不改变 workflow 阶段、状态机或 S-009 已定义的状态呈现语义。

## 验收标准

1. 满足存在性、导航用途和生命周期条件的文档引用使用可点击且有意义的 Markdown 链接。
2. 本地链接从来源文档目录正确解析；章节引用只使用可确认的稳定锚点。
3. Manifest 和阶段记录需要审计身份时同时保存导航链接与精确仓库相对路径。
4. 未创建目标、命令参数、配置值、输出位置和机器读取的文件身份继续使用状态与精确路径文本。
5. `docs/` 与 `build/` 的链接方向符合长期资产和运行记录的生命周期边界。
6. 移动或重命名后不存在遗留旧链接；失效本地目标会阻止提交或 Completion 声明。
7. 持久化文档不包含本机绝对路径、`file://` 或编辑器专用 URI。
8. 四个已实现 workflow 对共享规则的接入保持对称，完整链接契约只在 `document-conventions` 中维护。
9. 契约测试覆盖选择性链接、路径身份保留、生命周期方向、目标存在性和禁止 URI，且不锁死具体显示名称。

## 假设与开放问题

- Story 的 Open Questions 为“无”，需求阶段没有阻塞问题。
- 是否新增实际 Markdown 链接检查脚本、是否校验 GitHub 风格标题锚点，属于技术方案选择，不改变上述验收目标。
- 本任务不要求批量修复无关历史文档；仅更新本次规则接入所需的权威资产、测试和直接示例。
- 本任务影响可安装包和用户可见 workflow 规则，实施阶段必须评估并更新根目录 `version`；若不更新，最终说明理由。
